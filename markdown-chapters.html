Another chapter

First of all, we need a container for the dialog to live in, we use a <div> element, because it doesn't have any semantic meaning itself. We explain that the <div> actually is a dialog by adding role="dialog".

Roles are defined by ARIA, which allows us to express the meaning of an element.

We also add tabindex="-1" to allow the dialog container to be focused, something we'll revisit soon. Because dialogs are not always visible, we also add.

To structure the content rendered by the dialog, we'll use <header> for the introduction, <section> for the content and <footer> for the buttons to interact with the dialog.

<div role="dialog" tabindex="-1" hidden> <header></header> <section></section> <footer></footer> </div>
